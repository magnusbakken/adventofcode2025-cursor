# Prompt: 2025-12-03

## User Request

Add Cursor instruction files using the .cursor/rules directory structure with .mdc files. Use the guidelines on https://cursor.com/docs/context/rules, with the standard mdc file anatomy.

For the F# rules, scope them so they're only applied when editing F# files.

General rules:
- Include the prompt used to create each change in a top-level directory called prompts/. Each file should be a markdown file named with the current date/time in ISO format.
- Give succinct answers.
- At the end of each response, add the emoji ðŸ¤– to make it clear that the rules are being picked up.

Language specific rules:
- Use F# for all the tasks.
- Employ F# best practices. Prefer writing code in a pure functional style whenever possible.
- Prefer terse code, without deliberately being obfuscating.

Advent of code rules:
- Name each file dayNN.fsx, where NN is the 0-prefixed day number, starting with day01.fsx.
- Each day of Advent of Code has two parts. You will need to solve part 1 before being shown part 2.
- Each task will be given in a file called e.g. day01_part1.txt, day01_part2.txt, day02_part1.txt, etc.
- Each day has an associated input file, which will always be called day01_input.txt, day02_input.txt, etc.
- The output of each task should simply be a line of console output for part 1, and then another one for part 2.

## Implementation

Created three `.mdc` files in `.cursor/rules/`:
1. `general.mdc` - General rules for all files
2. `fsharp.mdc` - F# specific rules, scoped to `.fs`, `.fsx`, and `.fsi` files using tags
3. `advent-of-code.mdc` - Advent of Code specific rules

Each file follows the standard MDC anatomy with frontmatter containing tags for scoping, followed by markdown content describing the rules.
